The Atmosphere Framework 2.1.0 released!

Cross-Framework and Cross-Browser Asynchronous Framework for Real Time Applications on the JVM

The Async-IO dev team are pleased to announce the release of The Atmosphere Framework 2.1.0! You can download it here

Besides all of the performance and easy of use improvements made between Atmosphere 2.0, version 2.1 comes with a host of improvements and new features. Improved jsr356, Improved CDI support, build in SocksJS support, zero copy bytes for WebSocket and close to 70 issues since 2.0! Those changes alone makes it worth upgrading!

Atmosphere 2.1.0: Stability, Performance and more territories supported!

I/O Frameworks and Servers

Tomcat all versions (including 8.0.x!), Jetty all versions (including 9.1.x), Undertow (all versions), WebLogic 12.x (websocket natively supported. Netty 3.x, Vert.x 2.x and Play 2.2.x

Scary number of Frameworks supported

Atmosphere 2.1 is natively supported in PrimeFaces, Wicket, Grails, Spring's Reactor, GWT, Jersey, Scalatra, Vaadin, RichFaces, Apache Tuscany, Apache Tapestry etc.

Atmosphere 2.1 also can be used with any Servlet based framework like RestEasy, Apache CFX, Apache Wink, framework like Akka, Cometd, Socket.IO, distributed solutions like Hazelcast, Redis, JGroups, JMS for extreme scaling!

Google 'Atmosphere + framework_name' for more information.

The Atmosphere Framework 2.1.Stack contains the following components

  • runtime: server side components for Servlet based framework.
  • javascript: client side library for Atmosphere
  • wAsync: Java client side library for Atmosphere
  • NettoSphere: server side server running on top of the Netty Framework
  • VertoSphere: server side server running on top of the Vert.x Framework
  • Playtosphere: server side server running on top of the Play! Framework
